namespace 命名空间

  1. 用于解决名称冲突问题
  2. 必须在全局作用域下声明
  3. 命名空间下可以放入函数、变量、结构体、类
  4. 命名空间可以嵌套命名空间
  5. 命名空间是开放的，可以随时加入新的成员
  6. 匿名命名空间
  7. 命名空间可以起别名

using使用

using声明

using 命名空间名::成员；

如果局部范围内还有该成员，会出现二义性问题，要注意避免

using编译指令

using namespace 命名空间名;

如果局部范围还有该命名空间内的成员相同名字的成员，则使用后者

C++对C语言的增强

  1. 全局变量检测增强

  2. 函数检测增强

    参数类型检测增强，返回值检测增强、函数调用参数检测增强

  3. 类型转换检测增强

  4. 结构体增强

    可以放入函数，使用时可以不加struct关键字

  5. bool类型增强

    C语言中没有

  6. 三目运算符增强

    C中返回的是值，C++中返回的是变量

  7. const关键字增强

    C语言中，const修饰的变量是伪常量，编译器会分配内存

    C++中，const会放入符号表中

    C语言中的const默认是外部链接，而C++中是内部链接

    const分配内存情况

    ​ ①对变量取地址会分配临时内存

    ​ ②extern编译器也会给const变量分配内存

    ​ ③用普通变量初始化const的变量

    ​ ④自定义数据类型，加const也会分配内存

    尽量以const替换#define

    ​ ①const有类型

    ​ ②const有作用域

  8. 引用

    1. 引用基本语法

      Type &别名 = 原名

      int a = 10;

      int &b = a;

    2. 引用必须初始化，初始化后不可以修改

    3. 对数组建立引用

      ①int arr[10];

      ​ int (&pArr)[10] = arr;

      ②typedef int(ARRAYREF)[10];

      ​ ARRAYREF &pArr2 = arr;

    4. 参数的传递方式

      1. 值传递
      2. 地址传递
      3. 引用传递

    5. 引用的注意事项

      1. 引用必须引用一块合法的内存空间，不能有NULL引用
      2. 不要返回一个局部变量的引用
      3. 如果函数的返回值是引用，那么这个函数的调用可以作为左值
      4. 引用的本质再C++内部实现是一个指针常量

    6. 引用的使用

      1. 指针的引用

        用一级指针引用可以代替二级指针

        struct Person

        {

        };

        fun(Person * &a);

      2. 常量的引用

        const int &ref = 10;

        加入const后， 编译器处理方式为： int tmp = 10， const int &ref = 10；

        但可以通过指针修改

        int *p = (int *)&ref;

        *p = 1000;

        常量引用使用场景 用来修饰形参为只读
